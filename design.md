# Panic Buffet — 設計書 (design.md)

食べ放題レストランの店長になって、3分間お客の波をさばくブラウザ向けアーケードゲーム。
Vite + TypeScript + Canvas で動く、依存ゼロの軽量プロトタイプ。

> ポートフォリオ用。**ぱっと見で「ちゃんと作ってる」と伝わる**ビジュアルと手触りを最優先する。
> 画像・音素材は使わず、絵文字＋図形＋CSS/Canvasアニメで魅せる。

---

## 1. コンセプト

- プレイヤー = 食べ放題ビュッフェの店長。
- 3種の料理 **🍕ピザ / 🍝パスタ / 🥤ドリンク** の在庫を切らさないように補充し続ける。
- お客が列に並び、料理を要求する。在庫があれば提供 → 満足 → スコア＆お金。
- 在庫切れで待たせるとお客の我慢ゲージが減り、キレて帰る → ペナルティ。
- **180秒**サバイブしてスコアを稼ぐ。終了後に結果画面＋ランク。

## 2. 画面構成

横長 1 画面（レスポンシブ、最大幅 960px、中央寄せ）。背景はやわらかいグラデーション。

```
┌──────────────────────────────────────────────┐
│  HUD:  ⏱ 残り時間   💰 スコア   🔥 コンボ   🏆最高 │
├──────────────────────────────────────────────┤
│                                              │
│   [ Canvas プレイエリア ]                     │
│   ・上部：並んでいるお客（絵文字）＋吹き出しの要求料理 │
│   ・お客の頭上に我慢ゲージ（細いバー）          │
│   ・提供成功/失敗時にスコアのポップアップが飛ぶ     │
│                                              │
├──────────────────────────────────────────────┤
│  在庫パネル（3つ横並び）                        │
│   🍕 [████████░░] 80%   [ 補充 ]              │
│   🍝 [███░░░░░░░] 30%   [ 補充 ]              │
│   🥤 [█████░░░░░] 50%   [ 補充 ]              │
└──────────────────────────────────────────────┘
```

3 つの状態を持つシングルページ:
- **Title**: タイトル、遊び方、最高スコア、[ START ] ボタン。
- **Playing**: 上記レイアウトでゲーム進行。
- **Result**: 最終スコア、ランク(S/A/B/C/D)、最高更新の演出、[ もう一回 ] ボタン。

## 3. ゲームルール

### 在庫 (Stock)
- 各料理 0〜100。開始時は全て 100。
- お客 1 人を提供するごとに **20** 消費。
- **補充ボタン / キー**で +40。ただし料理ごとに **0.8秒のクールダウン**（連打防止、ボタンが一瞬無効化＆視覚フィードバック）。
- キー操作: `1`=ピザ補充, `2`=パスタ補充, `3`=ドリンク補充。

### お客 (Customer)
- 一定間隔でスポーン。間隔は時間経過で短くなる（序盤 1.8秒 → 終盤 0.7秒へ線形で詰める）。
- 同時に並べる最大人数 = 5。満員時はスポーンを待つ。
- 各お客はランダムに 1 種の料理を要求（吹き出しで表示）。
- 我慢ゲージ = 100 から毎秒減少（約 -14/s、満員でなくても減る）。
- 列の先頭のお客の要求料理に**在庫が 20 以上ある状態**で、プレイヤーが提供操作 or 自動提供されると成功。
  - 提供方式は**自動**：先頭客の要求料理の在庫が足りていれば一定の「調理時間」(0.6秒)後に自動で提供される。プレイヤーの仕事は在庫を切らさないこと＝補充に集中させる。
- 在庫不足のまま我慢ゲージが 0 → **キレて退店**（怒り演出、ペナルティ）。

### スコア / コンボ
- 提供成功: **+100 × コンボ倍率**。さらに我慢ゲージが高いほどボーナス（満タン提供で +50）。
- コンボ: 連続提供成功で 1 ずつ増加、倍率 = `1 + combo*0.1`（上限 3.0）。失客でコンボ 0 にリセット。
- 失客: **-150**、コンボリセット。スコアは 0 未満にはしない。

### 終了
- 180 秒経過で終了 → Result。
- ランク閾値（目安、調整可）: S ≥ 6000 / A ≥ 4000 / B ≥ 2500 / C ≥ 1200 / D それ未満。
- 最高スコアは `localStorage` キー `panicbuffet.highscore` に保存。Result で「最高更新！」演出。

## 4. 手触り / 演出（ここが本体）

- 提供成功: お客が `😋`→ ぴょこっと跳ねて消える、`+100` が上にフロートしてフェード、コンボ表示が拡大バウンド。
- 失客: お客が `😡` に変わり画面が一瞬赤くフラッシュ＆軽くシェイク、`-150` がフロート。
- 在庫バー: 値変化をイージングで滑らかに補間。残量 25% 以下で赤く点滅して警告。
- 補充ボタン: 押下でスケール、クールダウン中はグレーアウト＋残りクールダウンの円形/バー表示。
- HUD の残り時間: 残り 10 秒で数字が赤く拍動。
- 背景: ゆるやかに動くグラデーション or 控えめなパララックス。やりすぎない。
- 全体のフォントは丸ゴシック系（`system-ui` ベース）、角丸・ソフトシャドウで今風に。

## 5. 技術設計

- **描画**: プレイエリアは `<canvas>`（DevicePixelRatio 対応でくっきり）。HUD・在庫パネル・ボタン・タイトル/結果は DOM + CSS（その方がアニメと操作が楽）。
- **ループ**: `requestAnimationFrame`。`update(dt)` と `render()` を分離。`dt` は前フレームからの経過秒で計算（フレームレート非依存）。
- **状態管理**: 純粋なデータ構造 `GameState` を 1 つ持ち、`update` がそれを変える / `render` が読むだけ、に徹する。グローバル可変状態を散らさない。
- **画面遷移**: `scene: 'title' | 'playing' | 'result'` を state に持たせて切替。

### ファイル構成（docs.md の計画に沿う）
```
index.html            … #app だけ。canvas/HUD は TS で生成
src/main.ts           … エントリ。シーン管理＋ゲームループ起動
src/style.css         … 全 UI スタイル（ViteテンプレCSSは破棄して書き直す）
src/game/types.ts     … GameState, Customer, Food などの型定義
src/game/state.ts     … 初期状態生成、定数(CONFIG)、localStorage I/O
src/game/update.ts    … 1フレーム分のロジック更新 update(state, dt, input)
src/game/render.ts    … canvas 描画 + DOM HUD 反映
src/game/index.ts     … 上記の再エクスポート
```

### 不要になるもの（削除）
- `src/counter.ts`（Viteテンプレの残骸）
- `src/assets/` の Vite/TS ロゴ（`hero.png` 等、未使用なら削除可）

## 6. 受け入れ条件（Done の定義）

- `npm install && npm run dev` で起動し、Title → Playing → Result が一通り遊べる。
- `npm run build`（`tsc && vite build`）が型エラー無しで通る。
- 在庫補充・客のさばき・コンボ・3分タイマー・結果＆ランク・最高スコア保存がすべて動く。
- 30 秒触っただけで「お、ちゃんとしたゲームだ」と感じる演出が入っている。
- 追加の外部ライブラリ無し（Vite/TS のみ）。画像/音素材なし。

## 7. やらない（今回スコープ外・必要なら相談）

- バックエンド / オンラインランキング / 外部API
- 3D 化、画像・音素材の追加
- 複数ステージ、アップグレード等の拡張要素
