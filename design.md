# Panic Buffet — 設計書 (design.md)

食べ放題レストランの店長になって、2分間お客の波をさばくブラウザ向けアーケードゲーム。
Vite + TypeScript + Canvas で動く、依存ゼロの軽量プロトタイプ。

> ポートフォリオ用。**ぱっと見で「ちゃんと作ってる」と伝わる**ビジュアルと手触りを最優先する。
> 画像・音素材は使わず、絵文字＋図形＋CSS/Canvasアニメで魅せる。

> このドキュメントは現在の実装に追従する。数値の正本は [`src/game/state.ts`](src/game/state.ts) の `CONFIG` と `TUNINGS`。

---

## 1. コンセプト

- プレイヤー = 食べ放題ビュッフェの店長。
- 3種の料理 **🍕ピザ / 🍝パスタ / 🥤ドリンク** の在庫を切らさないように補充し続ける。
- お客が列に並び、料理を要求する。在庫があれば自動提供 → 満足 → スコア＆コンボ。
- 在庫切れで待たせるとお客の我慢ゲージが減り、キレて帰る → ペナルティ。
- ビュッフェなので **待っている客は自分で取って食べ、在庫が勝手に減る**（つまみ食い）。先頭だけでなく全ラインに目を配る必要がある。
- **120秒**サバイブしてスコアを稼ぐ。終了後に結果画面＋ランク。
- 開始前に **難易度（Easy / Normal / Hard）** を選ぶ。

## 2. 画面構成

横長 1 画面（レスポンシブ、最大幅 960px、中央寄せ）。背景はやわらかいグラデーション。
プレイ画面は縦に 1 画面へ収まるよう、Canvas が残りの高さに自動フィットする。

```
┌──────────────────────────────────────────────┐
│  HUD:  ⏱ 残り時間   💰 スコア   🔥 コンボ   🏆最高 │
├──────────────────────────────────────────────┤
│                                              │
│   [ Canvas プレイエリア ]                     │
│   ・上部：並んでいるお客（絵文字）＋吹き出しの要求料理 │
│   ・お客の頭上に我慢ゲージ（細いバー）          │
│   ・提供成功/失敗時にスコアのポップアップが飛ぶ     │
│   ・失客は 😡 が浮き上がってフェードアウト         │
│                                              │
├──────────────────────────────────────────────┤
│  調理中バー（先頭客の自動提供の進捗）             │
│  在庫パネル（3つ並び）                          │
│   🍕 [████████░░] 80%   [ 補充 ]              │
│   🍝 [███░░░░░░░] 30%   [ 補充 ]              │
│   🥤 [█████░░░░░] 50%   [ 補充 ]              │
└──────────────────────────────────────────────┘
```

3 つの状態を持つシングルページ:
- **Title**: タイトル、遊び方、**難易度セレクタ（Easy/Normal/Hard）＋説明文**、その難易度の最高スコア、[ START ] ボタン。
- **Playing**: 上記レイアウトでゲーム進行。
- **Result**: 最終スコア、ランク(S/A/B/C/D)、難易度表示、最高更新の演出、[ もう一回 ] ボタン。

## 3. ゲームルール

数値は難易度で変わる。下表は `TUNINGS`（state.ts）の正値。難易度に依存しない不変値は `CONFIG`。

### 難易度別パラメータ

| 項目 | 🍵 Easy | 🍙 Normal | 🌶️ Hard |
|---|---|---|---|
| 開始在庫 | 100 | 80 | 60 |
| 提供コスト（1人あたり消費） | 20 | 22 | 24 |
| 補充量 | +45 | +34 | +34 |
| 補充クールダウン | 0.7秒 | 0.8秒 | 0.62秒 |
| キッチン | 各ライン独立 | 各ライン独立 | **共有（同時に1品のみ）** |
| 調理時間（自動提供） | 0.65秒 | 0.58秒 | 0.52秒 |
| 我慢の減り（基準/秒） | 12 | 16 | 21 |
| 我慢の減りの終盤加速 | ×1.3 | ×1.5 | ×1.7 |
| つまみ食い（待機客が消費/秒） | 0 | 3 | 6 |
| スポーン間隔（序盤→終盤） | 2.0→0.9秒 | 1.6→0.65秒 | 1.3→0.45秒 |
| 失客ペナルティ | -100 | -150 | -250 |
| ランク S / A / B / C | 26k/19k/12k/6k | 35k/25k/15k/7k | 40k/29k/18k/8k |

### 不変値 (CONFIG)
- ゲーム時間 **120秒**、同時に並べる最大人数 **5**。
- 提供基礎点 **100**、コンボ係数 **0.1/連続**、コンボ倍率上限 **3.0**、満タン提供ボーナス上限 **+50**。
- 在庫警告しきい値 **30%以下**。

### 在庫 (Stock)
- 各料理 0〜100。開始時は全て `startStock`。
- お客 1 人を提供するごとに `serveCost` 消費。
- **待機中のお客は自分の要求料理を `nibblePerSecond`/秒 ずつ消費**（つまみ食い）。複数ラインが同時に減るので全体管理が要る。
- **補充ボタン / キー**で `+refillAmount`。`refillCooldown` のクールダウン中はボタンが無効化＆残量バー表示。
  - **Hard はキッチン共有**：3ライン共通の1本のクールダウン。1品補充すると、解除されるまで他の品も補充できない。複数ラインが同時に切れかけたとき、どこから埋めるかの取捨選択を毎秒迫られる。
- キー操作: `1`=ピザ補充, `2`=パスタ補充, `3`=ドリンク補充。

### お客 (Customer)
- 一定間隔でスポーン。間隔は時間経過で `spawnStart → spawnEnd` へ線形に詰まる。
- 同時に並べる最大人数 = 5。満員時はスポーンを待つ。
- 各お客はランダムに 1 種の料理を要求（吹き出しで表示）。
- 我慢ゲージ = 100 から毎秒減少。減少量は終盤ほど増える: `patienceDrainPerSecond × (1 + patienceDrainRamp × 経過率)`。
- 提供方式は**自動**：列の先頭客の要求料理の在庫が `serveCost` 以上あれば、`cookTime`（調理時間）後に自動で提供。プレイヤーの仕事は在庫を切らさないこと＝補充に集中。
- 在庫不足のまま我慢ゲージが 0 → **キレて退店**（😡 演出、ペナルティ）。

### スコア / コンボ
- 提供成功: **+100 × コンボ倍率** ＋ 我慢ボーナス（提供時の我慢割合 × 最大50）。
- コンボ: 連続提供成功で 1 ずつ増加、倍率 = `1 + combo×0.1`（上限 3.0）。失客でコンボ 0 にリセット。
- 失客: `-lostPenalty`、コンボリセット。スコアは 0 未満にはしない。

### 終了
- 120 秒経過で終了 → Result。
- ランクは難易度別の `rankThresholds`（上表）で判定。
- 最高スコアは **難易度ごとに** `localStorage` 保存（キー `panicbuffet.highscore.<difficulty>`）。Result で「最高更新！」演出。

### 実績（やりこみ要素）
全12種。`src/game/achievements.ts` が正本。`localStorage` に2キーで永続化：`panicbuffet.stats`（通算統計＝総提供数・プレイ回数・最高コンボ）と `panicbuffet.achievements`（解除済みID配列）。プライベートブラウジング安全（全I/Oをtry/catch）。

- **ゲーム内（即時解除＋トースト）**: はじめての一皿（初提供）/ コンボスタート（コンボ×2＝連続10）/ 絶好調（コンボ×3＝連続20, 秘密）/ コンボマシン（1ゲームでコンボ30, 秘密）。
- **ゲーム終了時に評価**: ビッグスコア（1ゲーム25,000点）/ ノーミス完走（失客0, 秘密）。
- **通算（累積）**: 百皿シェフ（通算100提供）/ 千皿マスター（通算1,000提供, 秘密）/ 常連プレイヤー（通算10ゲーム）。
- **ランク系**: S ランク達成（いずれかの難易度でS）/ Hard S ランク（HardでS, 秘密）/ ダイヤモンドシェフ（Hardを失客0でクリア, 秘密）。
- **UI**: 解除時に右上から **トースト**（「実績解除！ 🌟 …」、約3.2秒で自動消滅）。タイトルに **「🏅 実績 N/12」パネル**（解除＝フルカラー、未解除＝グレー、秘密かつ未解除＝🔒/???）。Result に **その回に解除した実績** を一覧表示。

## 4. 手触り / 演出（ここが本体）

- 提供成功: 先頭客が `😋` に変わって跳ね、提供で消える。`+点数` が上にフロートしてフェード、コンボ表示が拡大バウンド。
- 失客: お客が `😡` になり画面が一瞬赤くフラッシュ＆軽くシェイク、`-点数` がフロート、😡 が浮き上がってフェードアウト。
- 在庫バー: 値変化をイージングで滑らかに補間。残量 30% 以下で赤く警告。
- 補充ボタン: 押下でスケール。クールダウン中はグレーアウト＋残りクールダウンのバー表示（Hard は共有クールダウンを全ボタンに反映）。
- HUD の残り時間: 残り 10 秒で数字が赤く拍動。コンボは加点時にバウンド。
- 調理中バー: 先頭客の自動提供の進捗を表示。
- 背景: ゆるやかに動くグラデーション＋淡い図形のパララックス。やりすぎない。
- 難易度セレクタ: 選択中をハイライト、下に一言説明（blurb）を表示。
- 全体のフォントは丸ゴシック系（`system-ui` ベース）、角丸・ソフトシャドウで今風に。

## 5. 技術設計

- **描画**: プレイエリアは `<canvas>`（DevicePixelRatio 対応でくっきり）。HUD・在庫パネル・ボタン・タイトル/結果は DOM + CSS（その方がアニメと操作が楽）。
- **ループ**: `requestAnimationFrame`。`update(dt)` と `render()` を分離。`dt` は前フレームからの経過秒（フレームレート非依存）。`window.advanceTime(ms)` で決定論的にステップ実行も可能（テスト/録画用）。
- **状態管理**: 純粋なデータ構造 `GameState` を 1 つ持ち、`update` がそれを変える / `render` が読むだけ、に徹する。難易度の現在値は `state.tuning` に解決して持つ。
- **画面遷移**: `scene: 'title' | 'playing' | 'result'` を state に持たせて切替。難易度選択は `input.selectedDifficulty` 経由で `state.difficulty / tuning` に反映。

### ファイル構成
```
index.html              … #app だけ。canvas/HUD は TS で生成
src/main.ts             … エントリ。シーン管理＋ゲームループ・入力
src/style.css           … 全 UI スタイル
src/game/types.ts       … GameState, Customer, Food, Difficulty, Tuning などの型
src/game/state.ts       … 初期状態生成、CONFIG（不変値）/ TUNINGS（難易度別）、localStorage I/O、ランク判定
src/game/update.ts      … 1フレーム分のロジック update(state, dt, input)
src/game/render.ts      … canvas 描画 + DOM 生成/反映（タイトル・HUD・在庫・結果・実績パネル/トースト）
src/game/achievements.ts … 実績定義・解除判定・通算統計の localStorage I/O
src/game/index.ts       … 上記の再エクスポート

docs/demo.gif           … 実プレイのデモGIF（README ヒーロー）
scripts/capture-demo.mjs    … ヘッドレスChrome+ffmpeg でデモGIFを録画（要 puppeteer-core, --no-save）
scripts/measure-difficulty.mjs … 純ロジックを Node で回し難易度バランスを計測（要 tsx, --no-save）
```

## 6. 受け入れ条件（Done の定義）

- `npm install && npm run dev` で起動し、Title（難易度選択含む）→ Playing → Result が一通り遊べる。
- `npm run build`（`tsc && vite build`）が型エラー無しで通る。
- 難易度選択・在庫補充・つまみ食い・客のさばき・コンボ・2分タイマー・結果＆ランク・難易度別最高スコア保存がすべて動く。
- 実績の解除（即時トースト／終了時評価）・タイトルの実績パネル・通算統計の localStorage 永続化が動く。
- Hard で共有キッチン（同時に1品のみ補充）が機能する。
- 30 秒触っただけで「お、ちゃんとしたゲームだ」と感じる演出が入っている。
- 追加の外部ランタイム依存なし（Vite/TS のみ）。画像/音素材なし。

## 7. やらない（今回スコープ外・必要なら相談）

- バックエンド / オンラインランキング / 外部API
- 3D 化、画像・音素材の追加
- 複数ステージ、アップグレード等の拡張要素
