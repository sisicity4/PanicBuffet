# プロジェクト進捗管理

## 目的
このドキュメントは、プロジェクトの進行状況とタスクの管理を行うためのものです。各セッションで作業した内容を記録し、次のステップに移る前に確認します。

## 作業ルール

1. **MVP要約**
   - 達成するまでPDCA（計画-実行-確認-行動）してセッションを終了します。
   
2. **使用技術**
   - Vite + TypeScript
   - HTML/CSS (Canvas/DOM)
   - localStorage

3. **ファイル構成**
   - package.json
   - index.html
   - src/main.ts
   - src/style.css
   - src/game/types.ts
   - src/game/state.ts
   - src/game/update.ts
   - src/game/render.ts
   - docs/progress.md

4. **実行コマンド**
   ```bash
   npm run dev
   ```

5. **最初に作るコード**
   - Vite + TypeScriptのプロジェクト作成
   - ゲーム画面
   - 料理在庫ゲージ
   - 補充ボタン
   - 簡単なゲームループ
   - 客の自動生成
   - 3分タイマー
   - 結果画面

## 相談が必要な条件

以下の場合は勝手に進めず、ユーザーに確認してください。

- デプロイ先を決めるとき
- 技術スタックを大きく変えるとき
- バックエンドを追加したくなったとき
- 外部APIを使いたくなったとき
- 画像素材や音素材が必要になったとき
- ゲーム仕様を大きく増やしたくなったとき
- 3D化したくなったとき
- セーブ機能を複雑化したくなったとき

## 実行中の手段と目的

「Vite + TypeScript」を使用して、ブラウザで遊べる「食べ放題レストラン経営ゲーム」の最小限の可動プロトタイプ（MVP）を作成します。

## 作業計画（3つ以内）

1. プロジェクト作成
2. Canvasベースの基本的なレイアウトとUI要素を描画する。
3. まずは、ピザ、パスタ、ドリンクの在庫ゲージのみ表示し、補充ボタンも実装します。

## 作業詳細（Do）

1. プロジェクト作成：`npm create vite my-app --template typescript`
2. Canvasベースの基本的なレイアウトとUI要素を描画する。
3. ピザ、パスタ、ドリンクの在庫ゲージのみ表示し、補充ボタンも実装します。

## チェック（Check）

1. プロジェクトが正しく作成されたかどうか
2. CanvasベースのレイアウトとUI要素が正常に描画されるかどうか
3. ピザ、パスタ、ドリンクの在庫ゲージが表示され、補充ボタンが機能しているかどうか

## 行動計画（Act）

MVPの完成にはまだ時間がかかりそうです。まずはこの段階で行えるだけ作り、その後に必要なものを徐々に追加していきます。
